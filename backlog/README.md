# Backlog

This folder contains documentation for issues that cannot be fixed immediately but need to be addressed in the future.

## Purpose

The backlog serves as a record of:
- Problems encountered during development
- Attempted solutions and their outcomes
- Current state of unresolved issues
- Suggestions for future investigation

## Issue Format

Each issue is documented in a numbered markdown file (e.g., `001-issue-name.md`) and includes:
- Problem description
- Attempted fixes
- Current state
- Future investigation suggestions
